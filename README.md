# AsmVm --- сборщик и исполнитель программ для учебной виртуальной машины

## Описание проекта

Данное приложение представляет собой **консольную утилиту**,
объединяющую функции ассемблера и интерпретатора для специально
созданной учебной виртуальной машины с простым текстовым языком команд.
С помощью программы можно:

-   Преобразовать исходный ассемблерный код в бинарный формат\
-   Выполнять полученные бинарные файлы на виртуальной машине\
-   Использовать операции работы с памятью (`load`, `read`, `write`)\
-   По желанию сохранять состояние памяти в формате XML после завершения
    выполнения

## Основные команды языка

| Команда | Формат | Описание |
|---------|--------|----------|
| **load**    | `load <dest> <const>` | Загружает константу в ячейку памяти `dest` |
| **read**    | `read <dest> <src> <offset>` | Считывает значение из памяти: `memory[dest] = memory[memory[src]+offset]` |
| **write**   | `write <dest> <src> <offset>` | Записывает значение из регистра/ячейки `src` в память: `memory[memory[dest]+offset] = memory[src]` |

## Особенности реализации

### 1. Assembler

-   Реализована поддержка всех доступных инструкций\
-   Каждая команда транслируется в бинарную структуру длиной 5 байт\
-   Выполняется проверка правильности аргументов и диапазонов адресов

### 2. Interpreter

-   Последовательно исполняет бинарные инструкции\
-   Обрабатывает косвенную адресацию для команд `read` и `write`\
-   Может отображать выполняемые операции и текущее состояние памяти\
-   Поддерживает создание XML-файла с дампом памяти

### 3. Дамп памяти (CSV)

-   Активируется отдельным параметром при выполнении команды `run`\
-   В дамп включаются только ячейки с ненулевыми значениями\
-   Пример структуры:

``` csv
Address, Value
0,5
1,10
2,15
3,20
4,25
```

### 4. Тестирование

-   Созданы примеры, демонстрирующие копирование данных между областями
    памяти\
-   Проверена корректность выполнения операций `load`, `read`, `write`\
-   Тестировался формат XML-дампа со всеми ненулевыми значениями памяти

### 5. Заключение

Проект реализует полный цикл:

**ассемблерный код → бинарный файл → выполнение → экспорт памяти**,

что делает его удобным инструментом для учебных задач и экспериментов с
виртуальной машиной.
